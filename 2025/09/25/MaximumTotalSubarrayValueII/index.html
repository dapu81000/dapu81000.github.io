LeetCode3691 - Maximum Total Subarray Value II

Description

給你一個整數陣列$nums$，長度為$n$，以及整數$k$，
定義一個子陣列 $nums[l…r]$ 的值為 $max(nums[l…r]) - min(nums[l…r])$
題目要求k個不完全重疊的子陣列的值的最大和
(完全重疊 => 左右端點都一樣)

Solution

觀察後可以發現，當我們固定子陣列的左端點時，假設為 $l$，則右端點為$l,l+1,…,n-1,n$時，這些值會是遞增的數列
基於這個性質，我們可以用一個max heap存值與當前左右界，假設為 $[l,r]$，接著再將下一個區間 $[l,r-1]$ 的資訊加入max heap中，然後找k次，就得到答案了
將區間加入max heap時，我們需要知道當前區間的值(最大減最小)，這部分則是可以用兩顆線段樹來維護(最大、最小)
整體的時間複雜度是 $O(k \log{n} \log{n})$，且 $k \leq min(10^5, \frac{n*(n+1)}{2})$

Follow Up

k的限制改為 $k \leq \frac{n*(n+1)}{2}$

Solution

首先先將題目拆解，我們先找出第 $k$ 大的值是多少
這部分可以使用二分搜來找，假設我們要找的值為 $x$ ，我們要算出值 $\gt x$的數量

再來是若將右端點固定時，假設為 $r$，左端點為 $1,2,3,…,r-1,r$時，數值是遞減的，概念跟前面一樣
接著我們可以使用單調隊列以及sliding window來維護當右端點為 $r$ 時，最左邊的點，使得這個點做為左端點時，這個區間的值 $\leq x$
這邊算第一個部分，時間複雜度為 $O(n\log U)$，$U$ 為陣列數值的值域

假設第k大的值為 $lower$
接著我們考慮兩個情況:

  1. 值 > $lower$
  2. 值 = $lower$

我們只要算出值>$lower$的部分，剩下的值=$lower$的部分可以直接用乘法相加

則假設我們現在固定住右端點為 $r$，且值 $\leq lower$ 的最左點為 $l$ ，對於右端點 $r$，我們要算出 $[1,r] + [2,r] + … + [l-1,r]$ 這些區間的值
區間和的部分，我們可以使用支援區間更新的線段樹來維護，假設我們要維護極大值，當前右端點為 $r$ 時，當前這個右端點可以更新到前面哪個的位置，找到位置後就區間更新為當前這個右端點的值，極小值也是同樣道理
而前面提到的符合條件的最左點 $l$ ，我們可以透過二分搜來找，算出當前區間的最大即最小值

最後我們將前述算法的$r$從$1$推到$n$，再算出總共加了幾次，再去加剩下的次數乘上lower，就完成了
整體的時間複雜度為 $O(n \log U + n \log n \log n)$

後記

這題關鍵的點在單調性的部分，就是固定左端點或右端點後會發現的性質，知道這個性質後第一個算法就能想到
不過第二個部分則是難度大幅提升@@，三四種知識點加在一起，需要對這些知識點都很熟練才有辦法想到和實作出來(我這部分完全看靈神的題解><(link: https://www.youtube.com/watch?v=jOuCDXIPVbk))

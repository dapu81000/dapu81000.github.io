LeetCode1381 - Design a Stack With Increment Operation

Description

題目要求你實作出一個stack
他可以

  • Push
  • Pop
  • Increment multiple elements from bottom

然後Pop的時候要順便輸出Pop掉的值

Solution1

可以發現題目給的限制都很小
所以可以直接一個一個加
更新的複雜度 $O(N)$

Solution2

但若當處理的數量${\leq}10^6$左右時，Solution1應該會超時
稍微想一下，會發現說其實前面的值沒必要先算，唯一要考慮的是當前stack的top要怎麼處理
Increment的時候去紀錄加到第幾個element及加了多少
Pop的時候算出原本的值加上前面紀錄的值，然後再更新Pop掉這個數對前面造成的影響
Push就不用另外多做處理
更新的複雜度 $O(1)$

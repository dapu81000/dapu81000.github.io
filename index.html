LeetCode1425 - Constrained Subsequence Sum

Description

給你一個序列（一串數字），找出一個子序列，使其滿足下列要求

  • 總和為所有可能子序列最大
  • 子序列內相鄰的數在原本的序列index的差不超過k

然後return該子序列的總和

Solution

應該很直覺可以用dp來做
先來定義狀態
$dp[i]$是在只有前$i$個數的情況下，有選取第$i$個數的最大子序列的和
寫成數學式的話大概長下面這樣

$dp[i] = {max_{j}}\{dp[j]\} + nums[i], \text{where } j{<}i \text{ and } i-j{\leq}k$

然後我們要求的就是這個dp陣列裡面的最大值

那我們要怎麼找$j$呢
首先分析一下，如果對於每個$i$，我都去爆搜可能的$j$
時間複雜度會是$O(n{\times}k)$
constraint是$1{\leq}k{\leq}n{\leq}10^5$，這樣做大概率會超時

那要怎麼快速地找$j$呢
這邊做法是把所有可能的$j$在做之前就存起來
那因為我們每次要挑的都是可能的$j$中dp值最大的
當前面可能的$j$超出範圍，那就要把這個j丟掉
最後我把我當前的dp值算出來後，如果前面可能的$j$的dp值比我小，就要丟掉，因為明顯剛算出來的這個值要馬被後面的用到，要馬就用到比這個值更大的dp值
總結就是有個單調性在，所以找$j$的複雜度可以降到$O(1)$
整體的時間複雜度就是$O(n)$

LeetCode2818 - Apply Operations to Maximize Score

Description

定義一個數字的Prime Score為該數字的質因數個數
例如 $300 = 2{\times}2{\times}3{\times}5{\times}5$, 則300的Prime Score為3

然後現在要你選k個不同的區間，使得每個區間中Prime Score最高的數相乘最大

Solution

首先是Prime Score的計算
可以一邊建質數表一邊算

再來是要算每個數字對應到的左右最大區間
使得這個數字是區間裡面Prime Score最大的數
直接一格一格左右推會超時 $O(n){\times}O(n)$
對於左區間來說
我們要的是前一個Prime Score比當前數字的Prime Score大的位子
觀察一下可以發現 前面的數字中 比當前數字的Prime Score還小的 對於後面還沒找的左區間沒有用
因為要是後面的Prime Score比當前的小 那左界最左也只會到當前
要是後面的Prime Score比當前的大 那就會是找前面比當前大的
那右區間也是相同道理
具有單調性 所以這樣整體的複雜度是 $O(n)$

找完每個數字對應到的左右區間
再來就是算答案
因為乘的次數不小
這邊要用到快速冪
這樣答案就出來了

LeetCode458 - Poor Pigs

2023/10/29 Update

Description

有很多桶子，每桶都有水，然後其中一桶有毒，豬豬喝了會死
每次測驗每隻豬豬都可以選定喝任意桶水
測驗次數最多n次
問你最至少要幾隻豬豬才能測出哪桶水有毒

Solution1

這題最關鍵的點應該就是要換個角度思考
她問你至少要幾隻豬豬才測得出來
那我們先找固定次數以及固定隻數的豬豬，最多可以測幾桶水
首先假設最多可以測$n$次，然後有$m$隻豬豬，
然後$dp[n][m]$是最多可以測幾桶水
下面歸類一下$dp[n][m]$的各項轉移式

測完一次後，

1.豬豬都沒死
豬豬數一樣，然後少一次機會測
$=> dp[n][m] += dp[n-1][m]$

2.死一隻豬豬
豬豬數少一，然後少一次機會測
注意到這裡要考慮到每隻豬不同，所以要乘上$\binom{m}{1}$
$=> dp[n][m] += \binom{m}{1} \cdot dp[n-1][m-1]$

3.死k隻豬豬
同理，選哪k隻豬豬死
$=> dp[n][m] += \binom{m}{k} \cdot dp[n-1][m-k]$

整理出轉移式之後，大概就可以把整個dp陣列求出來了
複雜度$O(n\cdot m^2)$
假設有k個桶子
就是找到最小的m使得$dp[n][m] \geq k$

Solution2

那如果當n,m越來越大，solution1就可能會超時
這個解法其實是建立在猜測的基礎上XD
觀察轉移式後，發現長得很像展開後的二項式，
所以這邊猜$dp[n][m] = (n+1)^m$
證明的話就用數學歸納法證，不難證就不放過程了(?

最後
假設有k個桶子，我們要找到最小的m
使得$k \leq (n+1)^m$
然後用log再移一下項，可以得到
${log(k) \over log(n+1)} \leq m$
然後ceiling一下，就找到我們要的$m$了
複雜度$O(1)$

Submit Code

LeetCode2334 - Subarray With Elements Greater Than Varying Threshold

Description

給你一串數字和一個threshold
問你能不能從這串數字找出一段連續的數字
使得這段數字都大於等於threshold除以長度

Solution

其實蠻直觀的
對於每個數字
找到前一個和下一個比他小的數字的位置
這樣就可以讓這個數字在這段數字中最小
有點greedy的味道
這裡可以用到monotonic stack來找前一個最小和下一個最小
這邊先上一下找前一個最小的code


大概長這樣
因為我要找前一個最小
所以比現在大的都沒用 因為後面要嘛選自己要嘛選到前面比自己還小的
這就是monotonic的原因
所以這邊可以達成$O(n)$找到前一個最小 下一個最小也是一樣
不過我一開始沒想出來 後來偷看Discuss才會==

然後為了方便計算長度 我找的是前一個最小的下一個數
即$[i-L,i+R]$這段連續數字,$nums[i]$是最小的數,然後讓這串數字盡量長，即讓$L$、$R$最大
